# Causes Atlas — Apple-version concept

A standalone exploration of the Jony-Ive reading of the cinematic
Causes Atlas surface. **Concept only.** This is not the live site and
nothing here is wired to production.

## View it

Double-click **`index.html`** — it opens in any browser, no build step.
React and Babel load from CDN; the component is transpiled in-browser.

Click the dark screen (or use arrow keys) to move through five scenes:

1. **Emergence** — the constellation weaves itself over ~3 seconds, then
   the manifesto fades in. No chrome, no buttons. Restraint as the
   opening drama.
2. **Drop** — the entire screen is the upload affordance. No "upload"
   button anywhere — drag a file over and the frame brightens.
3. **The graph reacts** — uploaded data ignites nodes inside the *same*
   constellation. The report is not a separate page; it is the graph
   re-lit. Hover gives one line of text, no card.
4. **Findings** — each top action is its own full-screen typographic
   moment. Big number, gene, the single action sentence, and the source
   attached to the claim itself.
5. **Synthesis** — the clustered summary rises as a sheet from the
   bottom, hairline gold separator, cluster-load bars, verification
   stamp.

## Files

- `index.html` — self-contained, openable preview (component inline)
- `causes_atlas_apple.jsx` — clean React component (ES module) for a
  developer to drop into a real build/iterate on

## What this is vs. the live site

The live site (`living_graph.html`, deployed to Vercel) is the
denser "Kanye" surface — everything visible at once. This concept is
the "Apple" reading: the same emotional payload achieved through
*less* — gesture over chrome, typographic cadence over density, the
report dissolved into the graph itself.

Both are legitimate. This folder exists so the idea can be seen and
felt before any decision to build it for real.
